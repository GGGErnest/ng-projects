Configurations

Ng-Ui2Qp configurations

We know that every use case could be different and therefore we've provided some configurations that will help you to adjust the library feature to your own specific use cases.

Global configurations

Global configurations are the configurations that are provided when importing the UI2QpModule and are going to be used within the scope of the module that Ui2QpModule was imported.

Configuration Object

These are the available configurations and what they do:

autoUpdating

Groups settings specific to the auto-updating feature of the Library

replaceState

Defines whether updating the URL due to a change in the model adds or replaces the state in the browser's history. Check this for more info about this topic

logLevel

cryptoSecretKey

Secret key used in the "encrypted" serializer and deserializers. If not provided a default value will be used.

Default values

These are the default values for each configuration if not provided.

UI2QpRoot Configurations

Ui2QpRoot accepts some settings which define how it should behave. Those settings are the same that were provided when importing UI2QpModule.

UI2QpGroup Configurations

UI2QpGroup can be configured when creating an instance with the Ui2QpBuilder, following are those configurations:

qpName

It allows the renaming of the default Qp's name for this Ui2QpGroup instance.

validatorOrOpts and asyncValidators

They have the same function that the similar properties in the Angular's FormGroup class. Check Angular documentation to know more about them.

Ui2QpControl Configurations

Ui2QpControl accepts some configurations when creating an instance with the Ui2QpBuilder, those configurations are the following:

qpName

Qp name to be used for this Ui2QpControl.

validatorOrOpts and asyncValidators

They have the same function that the similar properties in the Angular's FormGroup class. Check Angular documentation to know more about them.

type

Type of the current Ui2QpControl. The type is used for assigning the right serializer and deserializers at creation time, if not provided the default is "string". Please checkout the serializer and a deserializerdocumentation to know more about how they affect the Ui2QpControl's value when set it or get it.

defaultVal

Value to used when is null the value to set.

We know that a Ui2QpControl instance binds a model property value to a Qp By default, if a qp value is null, it will be removed from the URL, that avoids useless URL segments such as ?q= (empty value). However, in some cases you might want this "default" to be a different value, e.g., a numeric input should perhaps default to 0 instead. Look at the next example.
